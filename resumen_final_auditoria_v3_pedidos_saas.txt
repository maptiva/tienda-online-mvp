### Checklist pre-merge

Antes de subir a producción, verificar:

1. Que el WARNING de discrepancia del caso 04 efectivamente apareció en los logs de
   Supabase (confirma que el canal de auditoría funciona en entorno real).

2. Que los tests se corrieron contra una base de datos de staging con datos similares
   a producción, no solo desarrollo local.

3. Que el script de REVOKE / GRANT de permisos se ejecuta como parte del deploy
   y no queda pendiente.

### Conclusión

Si esas tres cosas están confirmadas, está listo para subir. Fue un proceso de
auditoría limpio — de V1 con riesgos críticos abiertos a V3 production-ready en
tres iteraciones.

Evolución del proceso:

  V1  → Arquitectura base con RPC seguro. Riesgos críticos abiertos:
          manipulación de precios, search_path, cross-store injection.

  V2  → Recálculo de totales en servidor, blindaje de search_path,
          validación de propiedad cruzada. Observaciones menores pendientes.

  V3  → Descuentos blindados, performance optimizada, logging interno,
          edge case de array vacío cubierto.

  V3.1 → 100% de casos de test aprobados. Production-ready.
