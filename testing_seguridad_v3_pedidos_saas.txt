# Sugerencias de Testing — Seguridad de Pedidos SaaS (V3)

La V3 quedó muy bien — incorpora todas las observaciones. Antes de subir a producción,
acá una guía de testing concreta:

---

### 🧪 Casos de Test Recomendados

**Happy path (deben retornar `success: true`)**

1. Pedido normal con productos válidos de la tienda, descuento en cero.
2. Pedido con descuento válido (menor al subtotal).
3. Pedido con múltiples ítems de la misma tienda.

**Seguridad — Manipulación de precios (deben retornar `success: false`)**

4. Enviar `p_client_total = 0.01` con ítems reales → el sistema debe aceptar el pedido
   pero usar el total calculado por el servidor. Verificar en la BD que el `total`
   guardado es el correcto.
5. Enviar `p_discount_applied = 99999` → debe bloquearse con "Cálculo de descuento no permitido".
6. Enviar `p_discount_applied = -10` → mismo bloqueo.

**Seguridad — Cross-store injection (deben retornar `success: false`)**

7. Tomar un `product_id` de la Tienda B e intentar insertarlo en un pedido de la Tienda A
   → debe retornar "Inconsistencia en catálogo de productos" y generar el RAISE WARNING en logs.

**Seguridad — Tienda inválida**

8. Enviar un `p_store_slug` que no existe → "Tienda no disponible".
9. Enviar un slug de una tienda con `is_active = false` → mismo resultado.

**Validación de payload**

10. Enviar `p_items = {}` (objeto en lugar de array) → "Formato de productos inválido".
11. Enviar `p_items = []` (array vacío) → el pedido se crearía con total 0, evaluar si
    querés agregar una guarda para arrays vacíos. (Ver nota abajo.)

**Auditoría — Logs**

12. Ejecutar el caso 4 y verificar en Supabase → Database → Logs que aparezca el WARNING
    de discrepancia de precios. Esto confirma que el logging silencioso funciona.

---

### 💡 Dos cosas que agregaría antes de mergear

El caso 11 (items vacío) es un edge case que la V3 no cubre explícitamente — un pedido
con `total = 0` podría insertarse sin error. Vale agregar:

    IF jsonb_array_length(p_items) = 0 THEN
      RETURN json_build_object('success', false, 'error', 'El pedido no tiene productos');
    END IF;

Y si el `CartModal.tsx` ya valida esto del lado del cliente, igual conviene tenerlo en
el servidor como defensa en profundidad.
